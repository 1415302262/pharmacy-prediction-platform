# 药学研究生复试项目报告（增强版）

## 题目

**基于真实公开数据的药物脂溶性预测：结构特征增强、传统机器学习、PyTorch 深度学习与集成学习比较研究**

---

## 一、研究背景

脂溶性（lipophilicity）是药物研发中的核心理化性质之一，与膜通透性、口服吸收、组织分布、溶解性和成药性密切相关。通过计算方法根据分子结构预测脂溶性，可以在药物研发早期降低实验成本、缩短筛选周期，因此具有明确的药学意义。

QSAR（Quantitative Structure–Activity/Property Relationship）方法的核心思想，是将分子结构转换为可计算的数字特征，再使用统计学习或机器学习模型预测目标性质。本项目以脂溶性为预测目标，建立了一个从真实数据下载、结构特征提取、模型训练、可视化分析到结果导出的完整流程。

与上一版相比，本次增强版进一步引入了更丰富的结构表征和多模型集成，使得模型性能与展示完整度都明显提升。

1. 使用真实公开数据构建一个适合药学复试展示的 QSAR 项目；

3. 比较传统机器学习、深度学习与集成学习表现；

## 三、数据来源与真实性说明

- 总分子数：`4200`

分层随机划分后：

- 训练集：`2939`
- 验证集：`630`
- 测试集：`631`

三部分分布接近，说明划分合理。

## 五、特征工程

### 5.1 基础理化描述符（13维）

包括：

- MolWt
- MolLogP
- TPSA
- HBD
- HBA
- RotBonds
- RingCount
- AromaticRings
- FractionCSP3
- HeavyAtomCount
- LabuteASA
- BalabanJ
- QED

这部分特征适合解释“模型从药学角度学到了什么”。

### 5.2 RDKit 全量2D描述符

项目先计算 RDKit 内置全量 2D 描述符，再在训练集内进行：

- 缺失列剔除
- 零方差列剔除
- 高相关列剔除

最终保留 **173 维描述符**。

### 5.3 Morgan Fingerprint

- Radius = 2
- Bits = 2048

该特征用于表达局部结构片段信息。

### 5.4 MACCS Keys

- 167维结构键编码

该特征进一步补充常见结构模式信息。

### 5.5 最终增强特征

增强版最终输入特征由三部分组成：

1. 筛选后的 RDKit 描述符
2. Morgan Fingerprint
3. MACCS Keys

这使得模型既能看到分子整体理化性质，也能看到结构片段模式。

## 六、模型设计

### 6.1 基线模型

1. `RF_basic_desc`
2. `SVR_basic_desc`
3. `XGB_baseline_desc_fp`

### 6.2 增强模型

1. `XGB_rich_desc_fp_maccs`
2. `CatBoost_rich_desc_fp_maccs`
3. `PyTorch_MLP_rich_desc_fp_maccs`
4. `Weighted_Ensemble_rich`

### 6.3 为什么集成学习有效

- XGBoost 更擅长非线性表格模式
- CatBoost 在复杂特征组合上很稳健
- PyTorch MLP 能学习连续高维特征的分布表示
- 三者误差模式不完全相同，因此加权集成往往优于单一模型

## 七、评估方法

采用三个指标：

- `R²`：越高越好
- `RMSE`：越低越好
- `MAE`：越低越好

主要关注测试集表现，以评价泛化能力。

### 8.2 相比上一版的提升

| 版本 | 最佳模型 | R² | RMSE | MAE |
|---|---|---:|---:|---:|

- `R² +0.0397`

### 8.3 结果解释

2. **三种增强模型表现接近**：说明升级方案稳定，而不是偶然；
3. **集成模型最优**：说明 XGBoost、CatBoost 和 MLP 的信息具有互补性。

## 十、深度学习部分怎么讲

本项目使用了基于 PyTorch 的 MLP，而不是更重的 GNN，原因是：

1. 当前 `cptac` 环境已具备 PyTorch 和 GPU，可直接稳定运行；
2. 指纹 + 描述符输入的 MLP 易于复现，适合复试展示；
3. 当前任务中，MLP 测试集 `R² = 0.6967`，已经接近树模型；
4. 在此基础上再做加权集成，进一步得到最优结果。

这说明：

> **深度学习在这个项目中不是“噱头”，而是真正参与了性能提升。**

## 十一、复试时怎么讲这段经历

### 30秒版本

> 我做了一个基于真实公开药物数据的 QSAR 项目，任务是根据分子结构预测实验脂溶性。第一版使用基础描述符和分子指纹，第二版进一步加入 RDKit 全量描述符、MACCS 结构键，以及 XGBoost、CatBoost 和基于 PyTorch 的 MLP 模型做集成，最终把测试集 R² 从 0.683 提升到 0.723，RMSE 从 0.679 降到 0.635。同时我还补充了 scaffold 和化学空间分析，让项目既有预测性能，也有结构解释性。

### 1分钟版本

### 问：为什么这次效果会更好？
答：因为加入了更丰富的结构特征，并利用多模型集成捕捉互补信息。

### 问：为什么不用更复杂的 GNN？

### 问：结构分析部分有什么价值？
答：它让项目不只是“黑箱预测”，而是能从 scaffold、结构空间和代表性分子的角度解释结果，更贴近药学研究思路。

## 十三、严格验证（Repeated Scaffold Split）

为了提升项目的科研可信度，我进一步采用了 **5 次重复的 scaffold-aware 划分验证**。与普通随机划分相比，这种方式要求训练集和测试集尽量不共享相同骨架，因此更接近“新骨架外推”的真实应用场景。

需要强调的是：

> **scaffold split 的分数略低是正常现象，这不代表模型变差，而代表验证更严格。**

严格验证结果如下：

| 模型 | R² mean ± std | RMSE mean ± std | MAE mean ± std |
|---|---:|---:|---:|
| Weighted_Ensemble_scaffold | 0.7138 ± 0.0215 | 0.6489 ± 0.0296 | 0.4962 ± 0.0226 |
| PyTorch_MLP_rich_scaffold | 0.6945 ± 0.0261 | 0.6698 ± 0.0200 | 0.5111 ± 0.0187 |
| XGB_rich_scaffold | 0.6847 ± 0.0234 | 0.6811 ± 0.0347 | 0.5241 ± 0.0252 |
| CatBoost_rich_scaffold | 0.6750 ± 0.0234 | 0.6914 ± 0.0295 | 0.5313 ± 0.0201 |
| XGB_baseline_scaffold | 0.6472 ± 0.0311 | 0.7200 ± 0.0299 | 0.5519 ± 0.0234 |

这个结果很重要，因为它说明：

1. 在更严格的 scaffold 验证下，增强版模型依然稳定有效；
2. 增强特征 + 集成学习在严格设置下仍优于基线；
3. 该项目不仅“能跑高分”，也具备更接近科研论文的验证意识。

## 十五、平台化展示与部署

为了让项目不只停留在“离线代码 + 图表报告”，我进一步把它包装成了一个可公开访问的交互式网页平台。

### 15.1 平台目标

平台不是为了重新训练模型，而是为了：

1. 让老师或其他人可以直接输入一个分子进行体验；
2. 在线查看项目核心结果；
3. 通过 API 方式对接其他前端或系统；
4. 让项目从“研究代码”进一步升级为“作品级展示”。

### 15.2 平台架构

本项目平台采用：

- **Flask**：负责网页渲染与 JSON API
- **Docker**：用于 Hugging Face Spaces 部署
- **预训练模型加载**：在平台侧直接读取训练好的模型文件

平台目录为：

- `hf_space/app.py`
- `hf_space/templates/index.html`
- `hf_space/static/style.css`
- `hf_space/Dockerfile`
- `hf_space/assets/`

### 15.3 平台功能

当前平台支持：

- 输入 `SMILES` 进行单分子预测
- 返回集成模型脂溶性预测值
- 展示标准化结构图
- 展示局部正负 SHAP 特征
- 展示项目核心结果图
- 展示 scaffold 严格验证结果
- 提供 `POST /api/predict` JSON 接口

### 15.4 为什么平台化对复试有帮助

平台化最大的价值，不在于“网页更炫”，而在于它能向老师传达：

这对计算机背景的你尤其重要，因为它体现出：
